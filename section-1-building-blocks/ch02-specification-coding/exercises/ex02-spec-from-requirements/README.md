# Exercise 2.2: Spec from Requirements

## Goal
Convert a set of verbal/informal requirements into a testable specification.

## Instructions

1. Read `requirements.md` — these are messy, real-world requirements from a stakeholder
2. Write a clean specification using the template from Exercise 2.1
3. Identify: What's missing? What's ambiguous? What contradicts?
4. Build from your spec

## The Point

Stakeholders give you requirements. You give the model specifications. This exercise practices the translation between the two.
