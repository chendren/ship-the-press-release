# Expected Outcome

After 3-6 iterations, you should have a working CLI tool that:
- Accepts a URL argument
- Fetches the page content
- Produces a text summary

## What You Probably Experienced

1. **First response**: A generic fetch + summarize script — functional but not quite right
2. **Iterations 2-3**: Fixing edge cases the model missed (error handling, HTML parsing, output format)
3. **Iterations 4-6**: Tweaking behavior to match what you actually wanted vs. what you said

## The Key Insight

The gap between "what you said" and "what you meant" is the entire reason specification coding exists. Vibe coding works when that gap is small or when you're exploring. It fails when the gap is large and you don't realize it until iteration 6.

## Discussion

- How many iterations did you need?
- At what point did you realize you needed to be more specific?
- Could you have saved iterations by writing down what you wanted first?
