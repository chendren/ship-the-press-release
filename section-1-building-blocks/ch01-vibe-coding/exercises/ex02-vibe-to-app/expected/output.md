# Expected Outcome

A working small app built through 5-15 conversational iterations.

## Conversation Log Template

| # | Prompt Summary | Response Quality | Correction? |
|---|---------------|-----------------|-------------|
| 1 | Initial request | Functional but generic | No |
| 2 | "Add X feature" | Good | No |
| 3 | "No, I meant..." | Fixed | Yes — correction |
| ... | ... | ... | ... |

## Typical Metrics

- **Total prompts**: 8-15
- **Corrections**: 30-50% of prompts
- **Wasted iterations**: 2-4 (where output regressed or went sideways)

## The Takeaway

Vibe coding is real development. It produces real software. But its efficiency degrades as complexity increases, because each iteration carries the accumulated ambiguity of every prior conversation turn.
