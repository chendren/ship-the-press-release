# Appendix I: Glossary of Key Terms

| Term | Definition |
|------|-----------|
| **Vibe Coding** | Building with your mental model as specification — conversational, iterative |
| **Specification Coding** | Building from an explicit written specification — precise, testable |
| **Hybrid Mode** | Strategically switching between vibe and spec based on task requirements |
| **Working Backwards** | Writing the press release/end-state artifacts before building |
| **PR/FAQ** | Amazon-style press release and FAQ for a product that doesn't yet exist |
| **Steering Doc** | CLAUDE.md context file that tells the model about you, your project, and your conventions |
| **Hook** | Automated enforcement of a rule — fires on tool use events |
| **RALPH Loop** | Test-driven autonomous build: Read, Analyze, Layout, Produce, Harmonize |
| **Context Engineering** | Investing in structured project context rather than per-request prompt optimization |
| **Skill** | Modular package that extends an agent with domain-specific knowledge and tools |
| **Agent** | Autonomous AI system with defined role, capabilities, and boundaries |
| **Translation Pipeline** | Infrastructure enabling non-English developers to work effectively with AI tools |
| **Prompt Library** | Pre-built prompts in a developer's native language for common tasks |
| **Semantic Map** | Vocabulary alignment between English technical terms and target language equivalents |
| **Artifact** | Any concrete end-state document: PR/FAQ, wireframe, sample output, dialog transcript |
| **Test Matrix** | Comprehensive test plan mapping requirements to test cases with priorities |
| **Build Plan** | Ordered implementation plan with phases, tasks, dependencies, and validation gates |
