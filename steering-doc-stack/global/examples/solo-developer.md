# Example: Solo Developer

## Identity & Context
- Full-stack indie developer building SaaS products
- Mac (Apple Silicon), deploy to Vercel + Supabase
- Projects: invoice-app, personal-site, side-project-tracker

## Code Style Preferences
- TypeScript always
- Functional for utilities, classes only where state warrants it
- Tailwind CSS, no custom CSS unless absolutely necessary
- Short functions, single responsibility
- No defensive coding for impossible cases

## Workflow Preferences
- Commit after each feature, never mid-work
- Plan mode for anything non-trivial
- Default effort: medium

## Tools Available
- Supabase CLI for database
- Vercel CLI for deployment
- Playwright for E2E testing

## Communication Style
- Concise. Code first, explanation after.
- Use tables for comparisons.

## What NOT to Do
- Never add TODOs or stubs — finish the thing
- Never create documentation files unless asked
- Never suggest improvements I didn't ask for
