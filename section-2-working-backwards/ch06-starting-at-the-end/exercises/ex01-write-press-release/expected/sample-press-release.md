# Expected: A press release that passes the Two-Paragraph Test

A reader should be able to explain your product after reading the first two paragraphs. If they can't, the press release needs work.

## Quality Signals

- [ ] Specific numbers (not "fast" — "under 100ms")
- [ ] Named target customer (not "businesses" — "teams of 5-20 engineers")
- [ ] Concrete capabilities (not "powerful" — "sandboxed execution with scoped permissions")
- [ ] Real pricing (not "competitive" — "$49/month for 3 agents")
- [ ] Clear availability (not "coming soon" — "available today at example.com")

See [acmeclaw-prfaq/](/acmeclaw-prfaq/) for a complete example.
