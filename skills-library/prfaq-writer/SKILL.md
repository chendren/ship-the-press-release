# PR/FAQ Writer

Create press release and FAQ artifacts that define the end state before building.

## When to Use

- Starting a new product or major feature (Chapter 6)
- Defining what "done" looks like before writing code
- Aligning a team on product direction
- Creating the input artifact for the Working Backwards pipeline

## Workflow

1. **Name the product** — if you can't name it, you don't know what it is
2. **Write the headline** — one sentence that explains the value
3. **Write for the customer** — not for your team, not for investors
4. **Be specific** — numbers, not adjectives
5. **Write the FAQ** — start with the hardest questions
6. **Validate** — use the checklist in [templates/prfaq/](../../templates/prfaq/)
