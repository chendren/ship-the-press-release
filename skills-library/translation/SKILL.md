# Translation

Infrastructure for non-English developers to work effectively with AI coding tools.

## When to Use

- Translating documentation or README files (Chapter 18)
- Building a prompt library in a non-English language (Chapter 16)
- Evaluating model performance for specific languages (Chapter 17)
- Setting up a translation pipeline for a team (Chapter 18)

## Supported Approaches

1. **Prompt library** — pre-built prompts in the developer's language (Ch 16)
2. **Semantic maps** — vocabulary alignment between languages (Ch 16)
3. **Translation pipeline** — automated translation with glossary and quality checks (Ch 18)

## Reference Implementation

See [translation-pipeline/](../../translation-pipeline/) for the complete TypeScript implementation.

## Tips

- The language gap is real and measurable — see Chapter 15
- Prompt libraries eliminate the need to compose English prose
- Glossaries prevent technical term mistranslation
- Quality checks catch issues before they reach users
